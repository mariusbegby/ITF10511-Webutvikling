Media queries: Dynamisk tilpasning av innhold

Noen ganger har man behov for å tilpasse måten innholdet på nettsiden blir vist til brukeren, basert på hvilken enhet den vises på. Dette kan vi løse med media queries. Media queries tillater oss å skrive CSS basert på spesifikke tilstander, f.eks. skjermstørrelsen til brukeren.

Media queries, hva og hvordan

Media queries brukes når man skal endre måten innholdet blir vist på til brukeren, basert på noen spesifikke tilstander. Vi kan også målrette spesifikke mediatyper. Mediatypene som finnes er:

  • all - matcher alle mediatyper/enheter
  • screen - matcher enheter med en skjerm som kan vise innhold
  • print - matcher dokumenter som er klare for å printes ut
  • speech - matcher enheter som kan lese innholdet opp med lyd, f.eks. en skjermleser

Vi kan også målrette etter funksjoner. F.eks. kan vi spesifisere width eller height for å målrette etter spesifikke skjermstørrelser/oppløsning. Eller så kan vi ha egne stiler for om enheten er rotert i landskapsmodus eller portrettmodus med orientation. Det finnes mange slike funksjoner som nettleseren kan sjekke etter, og det gjør at vi kan lage den beste brukeropplevelsen, uansett hvilken enhet eller tilgjengelighet brukeren trenger.

Eksempel kode

Her er et eksempel på et par forskjellige media queries, som viser hvordan de fungerer: 

/* Standard stil, hvis ingen media query matcher */
body {
    background: red;
}

/* Minimum 600 piksler og over */
@media (min-width: 600px) {
    body {
        background: green;
    }
}

/* Minimum 1200 piksler og over */
/* Bare hvis siden blir vist på en skjerm */
@media screen and (min-width: 1200px) {
    body {
        background: blue;
    }
}

Dersom nettleservinduet er under 600 piksler, vil bakgrunnsfargen være rød, dette er "fallback" stilen. Hvis vinduet er over 600 piksler, men under 1200 piksler, vil bakgrunnsfargen være grønn, og hvis vinduet er over 1200 piksler vil bakgrunnsfargen være blå.

Fordeler, ulemper og bruksområder

Fordelene med media queries er mange. Med riktig bruk kan man tilby forskjellige utgaver av nettsiden basert på brukeren sin enhet, som sikrer best mulig brukeropplevelse. Det er ikke sikkert desktop opplevelsen er best på mobil, så da kan man bruke media queries til å endre på stilen for mobilvisning. Noen sider trenger kanskje også ekstra fokus på tilgjengelighet, og da kan man enda bedre tilpasse brukeropplevelsen for de med ekstra behov. Med media queries blir nettsidene enda mer dynamiske.

Når man skal lage forskjellige brukeropplevelser basert på oppløsning og mye annet, betyr dette også mer koding. Dette er en av de største ulempene med media queries; Du må bruke mer arbeid og tid for å lage nettsiden før den kan serveres til brukeren. Dette gjør det også vanskeligere å vedlikeholde koden. Skal du legge til noe nytt eller endre på designet, må man gjøre dette for alle brukeropplevelsene.

Skal man bruke media queries for å gjøre siden mer responsiv, burde man tenke "mobile first". Det vil si at man starter med å designe for mobilopplevelsen, og heller legger til media queries målrettet for større og større skjermstørrelser for tablet, desktop etc. Mobilversjonen av et nettsted vil alltid være brukbar på alle skjermstørrelser, så derfor er det bedre å gå denne veien, enn å designe for desktop versjonen først.
