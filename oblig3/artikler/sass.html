SASS: CSS med mindre kode og mer funksjonalitet

Det er ikke så morsomt så scrolle ned hundrevis av linjer for å finne den ene knappen du skal endre fargen på. SASS hjelper oss på mange måter å strukturere stilene til nettsiden vår, som gjør det enklere å opprettholde samt tilbyr mer funksjonalitet enn vanlig CSS.

Hva er forskjellen mellom SASS og CSS?

Preprosessering. Det som gjør SASS unikt er at det ikke er kode som kjører på nettsiden din. Istedenfor, må SASS koden vi skriver bli omgjort til vanlig CSS før det kan kjøres i nettleseren. Koden blir preprosessert og behandlet, gjort om til vanlig CSS før den leveres til brukeren. Nettleseren forstår nemlig ikke SASS kode. SASS er et verktøy vi kan bruke som kan gjøre livet vårt enklere og spare tid. Vi kan lage mer "avanserte" funksjoner og enklere benytte lik kode om igjen, disse funksjonene vi bruker blir kalkulert og prosessert før det gjøres om til CSS-

Hva gjør SASS bedre?

I SASS er det mange fine innebygde verktøy som vi kan ta i bruk. Blant disse er:

  • Variabler - enkelt gjenbruke verdier, f.eks. farger
  • Nesting - få en ryddigere struktur i koden
  • Partials - Små CSS snippets for gjenbruk
  • Modules - Slipp å ha alt i en fil
  • Mixins - Enkle funksjoner for generering av kode
  • Extend - For tagger som deler like properties
  • Operators - Bruk matematikk i koden

│
├───pages
│       _article-pages.scss
│       _main-page.scss
│
└───utilities
        _mixins.scss
        _variables.scss
